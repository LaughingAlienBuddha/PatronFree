ISSUE FIXED:
-----------
Username was hardcoded as "Aarav Mehta" or "DevX Team" across all pages and
dashboards. It wasn't changing based on the logged-in user.

SOLUTION IMPLEMENTED:
---------------------
Created a custom React hook `useUserProfile` that:
1. Listens to Firebase authentication state changes
2. Fetches the real user profile from the backend
3. Provides user data to all components
4. Automatically updates when user logs in/out
5. Falls back gracefully if profile not yet created

HOW IT WORKS:
-------------
1. User logs in via Firebase (Google/GitHub/Email)
2. useUserProfile hook detects auth state change
3. Hook fetches user profile: GET /profile (backend API)
4. Backend returns user from MongoDB database
5. All components receive real user data via hook
6. Components render with correct username

DATA FLOW:
----------
Firebase Auth (client)
        ↓
useUserProfile hook
        ↓
apiCall to backend
        ↓
Backend /profile endpoint
        ↓
MongoDB user collection
        ↓
Back to component
        ↓
Render with real username

BACKEND INTEGRATION:
--------------------
Required endpoint: GET /profile
- Must verify Firebase ID token
- Must return user profile from MongoDB
- Must include: id, name, email, profilePic, provider

Response format:
{
  "success": true,
  "data": {
    "id": "firebase_uid",
    "name": "User Name",
    "email": "user@example.com",
    "profilePic": "https://...",
    "username": "optional",
    "provider": "google|github|email"
  }
}
